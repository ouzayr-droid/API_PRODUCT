# Contribution

Merci pour votre intérêt à contribuer.

Processus

- Fork → Feature branch → Commit propre → PR vers `main`.
- Respecter le style C# du projet (nullable, conventions de nommage).
- Ajouter tests pour toute nouvelle logique métier.

Guide rapide

1. Créez une branche: `git checkout -b feat/my-change`
2. Implémentez et testez localement.
3. Ouvrez une Pull Request et décrivez les changements.

Contact

- Ouvrir une issue pour discuter des grosses modifications avant implémentation.
